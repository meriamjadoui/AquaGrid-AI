# Leak Detection Model — Training Metrics

This document provides a scientific evaluation of the Random Forest model used in the production ML pipeline.

## Model
- **Algorithm:** `RandomForestClassifier` (scikit-learn)
- **Trees:** 11 (`n_estimators`)
- **Max depth:** 6
- **Class weighting:** Balanced (leak events account for ~17% of ticks)

## Dataset
- **Source:** Synthetic, generated by `generate_data.py`.
- **Methodology:** Simulates realistic `flowRate` and `reservoirLevel` time series with injected leak events at random points in each session.
- **Total rows:** 24,000
- **Train/test split:** 80/20, stratified
- **Positive class (leak) ratio:** 16.87%

## Test Set Performance

> [!TIP]
> The model is optimized for **high recall**. In a water grid, a false positive (unnecessary maintenance check) is vastly cheaper than a false negative (missed leak leading to infrastructure damage).
