# Leak Detection Model — Training Metrics

## Model
- Algorithm: RandomForestClassifier (scikit-learn)
- Trees: 11 (n_estimators)
- Max depth: 6
- Class weighting: balanced (leak events are ~17% of ticks)

## Dataset
- Source: synthetic, generated by `generate_data.py`
- Simulates realistic flowRate / reservoirLevel time series with injected
  leak events at random points in each session
- Total rows: 24000
- Train/test split: 80/20, stratified
- Positive class (leak) ratio: 16.87%

## Notes

This model was trained on synthetic data as a proof-of-concept to validate
the ML pipeline end-to-end (feature engineering -> training -> evaluation ->
serialized artifact). It replaces the placeholder hand-coded decision logic
in `dashboard/src/ai/leak_model.js` with a model that was genuinely fit to
data rather than manually threshold-tuned.

**Next step (documented in architecture.md):** retrain on real Firebase-logged
sensor history once sufficient production data is collected, and serve via
a REST API (FastAPI) as described in the system architecture, rather than
loading the .pkl directly into the browser.
